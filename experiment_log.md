# 실험 로그

이 문서는 LG Aimers 8기 EXAONE 4 양자화 대회에서 수행된 실험의 시행착오와 실패 원인을 정리한 기록입니다. 각 실험은 번호별로 정리되어 있으며, 시도한 방법, 이유, 결과 점수, 실패/하락 원인, 교훈을 기록하였습니다.

## 실패한 실험 0: GPTQ + SmoothQuant 논문 적용
- **시도한 것**: GPTQ와 SmoothQuant를 결합하여 적용.
- **이유**: 양자화 효율성을 높이기 위해 두 기법을 결합.
- **결과 점수**: 실패 (점수 미기재).
- **실패/하락 원인**:
  1. GPTQ + AWQ 방식은 누적 양자화가 의미 없거나 데이콘에서 지원하지 않음.
  2. GPTQ는 calibration 데이터로 Hessian 근사를 통해 FP weight를 저비트로 변환.
  3. SmoothQuant는 weight뿐만 아니라 config.json 변경과 activation scale 적용이 필요하며, 데이콘 서버에서 커스텀 forward 로직을 지원하지 않음.
- **교훈**: 데이콘 환경에서는 표준 양자화 방식만 지원되며, 커스텀 연산은 불가능.

## 실험 1
- **시도한 것**: Calibration을 늘리고 샘플을 줄여 양자화 (GPTQ + AWQ).
- **이유**: 정확도 향상을 기대.
- **결과 점수**: 0.47
- **실패/하락 원인**: 정확도는 상승했으나 속도가 떨어져 점수 하락.
- **교훈**: 속도와 정확도의 균형이 중요.

## 실험 2
- **시도한 것**: GPTQ modifier에 group size 조정 (block size=128, desc_act=True), last_user_only 적용, 텍스트 길이 필터 (200 <= len(text) <= 2000), quantization_config 삭제 및 수동 수정.
- **이유**: 양자화 효율성 향상.
- **결과 점수**: 0.45
- **실패/하락 원인**:
  1. 입력 분포 변경으로 최적화가 아닌 분포 완전 변경.
  2. 텍스트 길이 제한으로 calibration 데이터에서 중요한 샘플 삭제.
  3. 수동 config 수정으로 메타데이터 불일치 및 비정상 경로 발생.
  4. W8A8이어도 calibration 분포 미스매치로 보호 방향 잘못 설정.
- **교훈**: Calibration 데이터 조작은 평가 분포와의 미스매치로 성능 저하 유발.

## 실험 3
- **시도한 것**: GPTQ 모델에 W8A8 양자화 적용, calibration 범위 64 <= x <= 512, seed 42.
- **이유**: 모델이 짧거나 긴 문맥에 취약하므로 중간 길이 텍스트로 평가 시 정확도 향상 기대, seed 변경으로 calibration set 구성 조정.
- **이유**: 모델이 짧거나 긴 문맥에 취약하므로 중간 길이 텍스트로 평가 시 정확도 향상 기대, seed 변경으로 calibration set 구성 조정.
- **결과 점수**: 0.61
- **실패/하락 원인**:
  1. Shuffle과 filter로 데이터셋 제한, 평가 분포와 미스매치.
  2. 짧은 입력이 많은 경우 calibration에서 제외되어 정확도 저하.
- **교훈**: 평가 분포가 블랙박스이므로 데이터셋 조작은 위험.

## 실험 4
- **시도한 것**: Filter, last_user_only, shuffle 제거, calibration 범위 [0:384]에서 [0:265]로 축소, 샘플 증가.
- **이유**: 0.62 모델 유지하며 샘플 증가로 미세 상승 기대.
- **결과 점수**: 미기재 (0.62 근처로 유추).
- **실패/하락 원인**: 미기재.
- **교훈**: 샘플 증가가 항상 긍정적이지 않음.

## 실험 5
- **시도한 것**: Ignore에 layer 0과 29 추가, 샘플 수 512로 증가.
- **이유**: Layer 0은 입력 임베딩 직후 민감, layer 29는 logit 직전 민감하여 오차 누적 방지.
- **결과 점수**: 0.59
- **실패/하락 원인**: Ignore로 FP16 레이어 증가 시 INT8 커널 미사용으로 속도 저하, SpeedNorm 영향.
- **교훈**: 보호 레이어제거 시 속도 손해 발생.

## 실험 6
- **시도한 것**: Hook을 이용해 outlier 분석, W8A8에서 W4A16으로 변경, ignore에 l28.up_proj 추가, calibration 512에서 256으로 감소.
- **이유**: Layer output과 proj output 분석으로 outlier 위치 파악 (L29 layer output, L28 up_proj), 속도 유지하며 성능 향상.
- **결과 점수**: 미기재 (최종 근처로 유추).
- **실패/하락 원인**: 미기재.
- **교훈**: Outlier 분석으로 특정 레이어 보호가 효과적, 샘플 수 조정은 환경에 따라 다름.

## 최종 코드로 이어진 결론
실험을 통해 GPTQ 기반 W4A16 양자화가 효과적임을 확인하였으며, outlier가 많이 발생하는 28번째 레이어의 up_proj를 보호하는 전략을 채택하였습니다. Calibration은 기본 설정 유지, 샘플 수 256개로 하여 안정성을 확보하였습니다.
